Ниже приведен обзор содержимого XML-документа:

Заголовок документа: Определяет версию XML (version="1.0") и
кодировку (encoding="UTF-8").

Пространства имен: Определяют пространства имен для использования в
XML-документе.
Пространство имен http://www.liquibase.org/xml/ns/dbchangelog
используется для элементов и атрибутов, определенных в Liquibase.

Заголовок документа:

<?xml version="1.0" encoding="UTF-8"?> - указывает версию XML и кодировку символов.
Корневой элемент databaseChangeLog:

xmlns="http://www.liquibase.org/xml/ns/dbchangelog" - определяет пространство имен для э
лементов и атрибутов внутри databaseChangeLog.
xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" - определяет
 пространство имен для XML
Schema Instance.
xsi:schemaLocation="http://www.liquibase.org/xml/ns/dbchangelog https://www.liquibase.org/xml/
ns/dbchangelog/dbchangelog-4.1.xsd" - указывает расположение XML Schema (XSD) для валидации
XML-документа.
Все последующие элементы и атрибуты внутри databaseChangeLog относятся к пространству имен
http://www.liquibase.org/xml/ns/dbchangelog.

Схема XSD: Определяет расположение схемы XSD для валидации XML-документа.

changeSet элементы: Представляют отдельные изменения базы данных.
Каждый changeSet имеет уникальный идентификатор (id) и указывает на
автора изменения (author).

insert элементы: Определяют операцию вставки данных в таблицу базы данных.
Каждый insert имеет указание на целевую таблицу (tableName) и столбцы
с их значениями.

column элементы: Определяют столбцы и значения, которые будут вставлены.
Атрибут name указывает на имя столбца, а содержимое элемента - на значение.
valueComputed атрибуты: Определяют вычисляемые значения для столбцов.
Вместо прямого указания значения, они используют функцию UUID_TO_BIN
для преобразования строкового представления UUID в двоичное представление.

Атрибуты author и id: Используются для идентификации автора и уникального
идентификатора каждого changeSet. Это помогает отслеживать историю
изменений базы данных.
