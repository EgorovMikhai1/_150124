Map: Данные из YAML часто собирают в Map или словарь,
особенно если YAML содержит именованные ключи и значения,
подобные JSON. Ключевым преимуществом использования Map
является то, что данные могут быть легко доступны по ключу,
что упрощает извлечение определенных значений.

Список или массив: Если данные YAML состоят преимущественно
из последовательностей, вы можете использовать список или массив.

Пользовательский объект (POJO): Если данные из YAML
представляют собой конкретный тип объекта в вашем приложении,
вы можете напрямую преобразовать данные YAML в этот объект.
